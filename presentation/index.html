Application architecture

MVC - well known architecture pattern

Application is divided into separated functional layers: Model, View and Controller
  • Model - stores data and sometimes data - related logic
  • View - presenatation layer, defines and performs view - related operations
  • Controller - business logic, connect other layers
MVC - schemat
Mutacje MVC - różne frameworki
  • Pure MVC (ExtJS)
  • MVVM (AngularJS)
  • MV* (Backbone)
Our mvc - schemat + opis Schemacik, kontroller, który ma referencję do widoku i ewentualnych modeli. Widok emituje eventy, na których nasłuchuje kontroller. Podobnie model. Kontroler wywołuje odpowiednie metody na widoku.

How to obtain the form values and pass them to controller?


var form = document.querySelector('.register-form');
form.addEventListener('submit', function (event) {
    event.preventDefault();
    var form = event.target,
        formData = new FormData(form);
    controller.fire('register', formData);
});
